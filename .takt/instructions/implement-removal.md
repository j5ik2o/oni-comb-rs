分析レポートに基づき、レガシーコードの削除を実行してください。

**やること:**

1. **分析レポートの読み込み**
   - `{report_dir}/00-legacy-analysis.md` を読み、削除計画を確認する

2. **段階的な削除の実行**
   - 高優先度（呼び出し元ゼロ）から順に削除する
   - 各削除ごとにビルド通過を確認する（`cargo build --workspace`）
   - 関連するテストも同時に削除/更新する
   - `#[allow(dead_code)]` 属性も削除する

3. **ビルド・テスト検証**
   - すべての削除完了後に `cargo test --workspace --all-targets` を実行
   - 失敗があれば原因を特定して修正する

4. **削除できなかった項目の報告**
   - 移行が必要で今回対応しなかった項目をリスト化する
   - 理由と今後のアクションを明記する

**注意:**
- 削除と新機能追加を混ぜない
- 公開API（`pub`）の削除は破壊的変更となるため、慎重に判断する
- **`pub` なシンボルは削除しない**（`pub trait` メソッド、`pub fn`、`pub type` 等は内部で未使用でも公開インターフェイスの一部）
- 公開シンボルの `dead_code` 警告は `#[allow(dead_code)]` で抑制する
- 不確実な項目は削除せず、レポートに記載する
